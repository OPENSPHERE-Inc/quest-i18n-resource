# リファレンス - 設定

---

## 1. 一般 - 基本プロフィール

- ①名前: （必須項目）サイト上で公開されるあなたのユーザー名
- ②言語: メインで使用する言語
- ③アクセスURL: SNSや、動画ch、配信ch のURLを10件まで設定できます。
  「<img src="/user-manual/plus-solid.svg" className="inline" height="12" /> フィールドの追加」を押してURL入力欄を増やせます。
  フィールドを削除する場合は「<img src="/user-manual/trash-alt-solid.svg" className="inline" height="12" />」を押してください。
- ④リクエストの受信を有効にする: クライアントからのリクエストを受け付ける場合はチェックを入れてください。

  ![Settings 001](/user-manual/ja/reference/settings-001.png)

---

## 2. 一般 - アバター

サイト上で、ユーザー名とともに表示されるアバター画像を設定してください。
画像は 2MB 以下の JPEG または PNG 形式のファイルでアップロード可能です。
また、アバター画像は画像に内接する正円でトリミングされます。

1. 「画像ファイルを選択...」ボタンを押して、アップロードする画像を選択してください。
2. 「保存」ボタンを押すと、選択した画像をアバターとしてアップロードできます。

   ![Settings 002](/user-manual/ja/reference/settings-002.png)

---

## 3. 一般 - 認証

1. **フォーム** へ以下のように入力してください。
   - ①メールアドレス: （必須項目）サインインや通知を受け取るためのメールアドレスの設定です。
   - ②パスワード: サインイン用のパスワードを変更する場合は入力してください。変更しない場合は空欄のままとしてください。
     パスワードには少なくとも6文字以上が必要です。
2. 「保存」ボタンを押してください。

   ![Settings 003](/user-manual/ja/reference/settings-003.png)

3. （最後にサインインをしたのが最近でない場合）続けて **再度サインインしてください** ダイアログが開きますので、
   （パスワードを変更する場合は、**変更前**の）パスワードを入力して「サインイン」ボタンを押してください。

   ![Settings 019](/user-manual/ja/reference/settings-019.png)

#### 確認用メールの送信

メールアドレスの確認を行うと、メールでの通知機能が利用できます。
「確認用メールの送信」ボタンを押して、送信されたメールに添付された確認URLにアクセスしてください。

尚、メールアドレスの確認は、メールアドレスが変更されるたびに必要となります。

![Settings 004](/user-manual/ja/reference/settings-004.png)

> メールアドレスの確認が済んでいる場合は、「確認用メールの送信」ボタンは表示されません。

## 4. 一般 - 通知

**メールによる通知** を設定するには、予めメールアドレスの確認を済ませる必要があります。
「確認用メールの送信」ボタンを押して、送信されたメールに添付された確認URLにアクセスしてください。

尚、メールアドレスの確認は、メールアドレスが変更されるたびに必要となります。

![Settings 005](/user-manual/ja/reference/settings-005.png)

> メールアドレスの確認が済んでいる場合は、「確認用メールの送信」ボタンは表示されません。

通知メールは `noreply@quest-bc.com` から送信され、
各項目で送信されるメールタイトル（Subject）は以下の通りです。

- ①フォロー中のユーザーがクエスト開始した時: `[QUEST] <パフォーマー名> がクエストを開始しました。`
- ②購読中のクエストへ新たなジャーナルが投稿された時: `[QUEST] <パフォーマー名> が新たなジャーナルを投稿しました。`
- ③リクエストを受信した時: `[QUEST](重要) <クライアント名> からリクエストが届きました。`
- ④ブーストしたクエストが中止された時: `[QUEST] クエストが中止されました。`
- ⑤ブーストしたクエストで投票開始した時: `[QUEST](アクションが必要) 投票が開始されました。`
- ⑥ブーストしたクエストが承認された時（最終判断の通知を含む）: `[QUEST] <パフォーマー名> のクエストが承認されました。` または `[QUEST](アクションが必要) 投票結果についてあなたの判断が必要です。`
- ⑦ブーストしたクエストが未承認のまま終了した時: `[QUEST] <パフォーマー名> は未承認のままクエストを終了しました。タイプAを除くブーストは返金されます。`
- ⑧投稿したディスカッションへ誰かが返信した時: `[QUEST] あなたのディスカッションに返信があります。`
- ⑨実行中のクエストがブーストされた時: `[QUEST] ブーストを受け取りました。`
- ⑩実行中のクエストが期限間近（あと1日）: `[QUEST](重要) 実行中のクエストが期限間近（あと1日）です。`
- ⑪実行中のクエストが承認された時: `[QUEST] クエストが承認されました。`
- ⑫実行中のクエストが未承認のまま終了した時: `[QUEST] クエストが終了しました。`
- ⑬実行中のクエストへディスカッションが投稿された時: `[QUEST] クエストにディスカッションが投稿されました。`
- ⑭あなたのオファーからクエストが開始された時: `[QUEST](重要) あなたのオファーに実行者が現れました!`

![Settings 006](/user-manual/ja/reference/settings-006.png)

## 5. 支払 - Symbolアカウント

本設定では、ブーストの支払いや、ブーストの受け取りに使用する Symbol アカウントの設定を行います。

冒頭で現在 QUEST に登録されている Symbol アカウントのアドレスを確認できます。

**Symbolアカウントの切り替え** フォームで、別の Symbol アカウントを登録できます。
詳しい手順は [ガイド - ブースト - ウォレットのセットアップ](/docs/guide/boost) を参照してください。

![Settings 007](/user-manual/ja/reference/settings-007.png)

## 6. 支払 - ウォレット

QUEST に搭載された簡易ウォレット機能に関するユーティリティが用意されています。

#### 秘密鍵のエクスポート（ブラウザウォレット有効の場合）

Symbolアカウントの設定でブラウザウォレットを有効にしている場合は、ブラウザローカルストレージに暗号化して保存された秘密鍵を復号し、表示できます。

1. 「秘密鍵をエクスポート」ボタンを押してください。

   ![Settings 008](/user-manual/ja/reference/settings-008.png)

2. **秘密鍵のエクスポート** ダイアログが表示されますので、「パスフレーズ」を入力し「秘密鍵を表示」ボタンを押してください。

   ![Settings 009](/user-manual/ja/reference/settings-009.png)

3. パスフレーズが正しければ秘密鍵が60秒間だけ表示されます。60秒経過すると非表示となり再びパスフレーズの入力を求められます。
4. 秘密鍵が表示されている間に「<img src="/user-manual/clone-solid.svg" className="inline" height="12" />」を押してコピーし、必要に応じてメモ帳等にペーストして保存してください。
5. 終わったら「閉じる」ボタンをおしてダイアログを閉じてください。

   ![Settings 010](/user-manual/ja/reference/settings-010.png)

> ブラウザウォレットが無効の場合は、「秘密鍵をエクスポート」ボタンが表示されません。

#### ブラウザウォレットの無効化と秘密鍵の削除

ブラウザローカルストレージに暗号化して保存された秘密鍵を削除し、ブラウザウォレットを無効化します。
ブラウザウォレットを無効化すると、ワンタイムウォレットへ自動的に切り替わります。

1. 「無効化と秘密鍵削除」ボタンを押してください。

   ![Settings 011](/user-manual/ja/reference/settings-011.png)

2. **秘密鍵削除の確認** ダイアログが表示されますので、このまま削除してよければ「OK」ボタンを押してください。

   ![Settings 012](/user-manual/ja/reference/settings-012.png)

3. ダイアログが閉じ、ブラウザウォレットが無効化されます。
4. 再び有効にするには「ブラウザウォレットを有効にする」ボタンを押してください（**Symbolアカウントの切り替え** フォームが開きます）

   ![Settings 018](/user-manual/ja/reference/settings-018.png)

> ブラウザウォレットが無効の場合は、「無効化と秘密鍵削除」ボタンが表示されません。

#### ウォレットへの入金

> 「ウォレットへの入金」とは、QUEST に登録した Symbol アカウントへの送金と同じ意味です。
> 以下の手順に従わずとも、単にアドレスへ送金すれば事足ります。
> ただし、**アドレスの間違いには注意してください。**

1. **入金** タブを開いてください。
2. 「金額」に入金したい金額を XYM で入力してください。
3. 金額を入力すると、その下にトランザクションURI または QR コードが生成されますので、手順に従ってデスクトップウォレットにインポートしてください。

   ![Settings 013](/user-manual/ja/reference/settings-013.png)

#### ウォレットからの出金

> 「ウォレットからの出金」とは、QUEST に登録した Symbol アカウントからの送金と同じ意味です。
> 以下の手順に従わずとも、単に当該アカウントからデスクトップウォレットなどを使って送金すれば事足ります。
> ただし、**秘密鍵を紛失すると誰にも（ユーザー自身も QUEST も）出金ができなくなるので注意してください。**

1. **出金** タブを開いてください。
2. 手順に従って秘密鍵をデスクトップウォレットにインポートしてください。
3. （必要に応じて）デスクトップウォレットを使って別の Symbol アカウントに送金してください。

   ![Settings 014](/user-manual/ja/reference/settings-014.png)

## 7. 支払 - ノードURL

ノードとは、Symbol ブロックチェーンにアクセスするための窓口となるサーバーの事を指します。ブロックチェーンは分散型システムですから、
アクセスするための窓口は誰でも自由に設置ができ、世界中の有志が数多くのノードを運営しています。

QUEST は幾つかのノードにアクセスするように予め設定されているので、特に本設定を触る必要はないのですが、自分や知り合いが運営するノードに接続したい、
といった場合は、こちらでノードを指定できます。

1. 「カスタムノードURLを使用する」にチェックを入れてください。
2. 「ノードURL」に `https://` から始まるノード URL を設定してください。
3. 「保存」ボタンを押してください。万が一指定したURLのノードが正常に機能しない場合はエラーが表示されます。

   ![Settings 015](/user-manual/ja/reference/settings-015.png)

## 8. 申請 - ユーザー削除

こちらの設定で QUEST からの退会を申請できます。

通常、退会の申請は自動的に処理され、退会が完了します。

1. **危険ゾーン** の「ユーザー削除する」ボタンを押してください。

   ![Settings 016](/user-manual/ja/reference/settings-016.png)

2. **ユーザー削除の確認** ダイアログが開きますので、このまま削除してよければ「OK」ボタンを押してください。

   ![Settings 017](/user-manual/ja/reference/settings-017.png)

3. （最後にサインインをしたのが最近でない場合）続けて **再度サインインしてください** ダイアログが開きますので、サインイン用のパスワードを入力して「サインイン」ボタンを押してください。

   ![Settings 019](/user-manual/ja/reference/settings-019.png)

4. **BASECAMP** に戻り、退会が完了します。

> 退会をするとユーザープロフィールは削除されアクセスできなくなりますが、既に作成済みのクエストやブースト、オファー、リクエスト、ディスカッション、ジャーナルは削除されません。
> また Symbol ブロックチェーンに書き込まれた取引情報も削除されません。
