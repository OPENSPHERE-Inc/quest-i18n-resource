# リファレンス - ダッシュボード

---

## 1. ダッシュボードへのアクセス

**DASHBOARD** へアクセスするには、**サイトヘッダー** の右端（モバイルブラウザの場合は左端）にあるアバターを押してください。

![Dashboard 008](/user-manual/ja/reference/dashboard-008.png)

---

## 2. ホーム

**ホーム** には購読しているクエストの新着ジャーナルと、
アクションが必要なクエスト、ブースト請求書、リクエスト、オファーが通知領域に表示されます。

#### 通知領域表示項目

- ①未払いのリクエスト: あなたが未だ支払っていないリクエストのリクエスト（初期ブースト）請求書
- ②未払いのブースト: あなたが未だ支払っていないブースト請求書
- ③投票実施中（のクエスト）: あなたが投票権を持つ、現在投票中のクエスト
- ④判断が必要（なクエスト）: あなたがブーストしたクエストで、授与か返金かの最終判断を待っているもの
- ⑤保留中のリクエスト: 下書き状態のリクエスト
- ⑥リクエスト受信箱: 受信したがまだ実行していないリクエスト
- ⑦実行中のクエスト: ブースト受付中のクエスト
- ⑧受付中のオファー: 応募受付中のオファー

#### ⑨ジャーナル

購読中のクエストの最新ジャーナルが最大20件表示されます。

#### ボタン

- ⑩「設定」ボタン: **Settings** ページを開きます。
- ⑪「サインアウト」ボタン: QUEST からサインアウトします。

![Dashboard 001](/user-manual/ja/reference/dashboard-001.png)

## 3. クエスト

**クエスト** にはパフォーマーとして作成したクエスト、クライアントとして作成したリクエストおよびオファーが最大 `10 件` 一覧されます。

「<img src="/user-manual/search-solid.svg" className="inline" height="12" /> もっと見る」を押すと、全ての一覧をリスト表示できます。

- ①あなたのクエスト: パフォーマーとして自身が作成、またはリクエストを受諾したクエストが最大 `10 件` 一覧されます。実行中のクエストは先頭に表示されます。
- ②あなたのリクエスト: クライアントとして自身が作成したリクエストが最大 `10 件` 一覧されます。
- ③あなたのオファー: プロモーターとして作成したオファーが最大 `10 件` 一覧されます。

![Dashboard 002](/user-manual/ja/reference/dashboard-002.png)

---

## 4. ブースト

**ブースト** にはパフォーマーとして受けたブースト、サポーターとして行ったブーストが一覧されます。
また、ブースト一覧をCSV形式でエクスポートできます。

「<img src="/user-manual/search-solid.svg" className="inline" height="12" /> もっと見る」を押すと、全ての一覧をリスト表示できます。

- ①かけられたブースト: パフォーマーとして自身のクエストに受けたブーストが最大10件一覧されます。
- ②かけたブースト: サポーターとして自身が行ったブーストが最大 `10 件` 一覧されます。

![Dashboard 003](/user-manual/ja/reference/dashboard-003.png)

#### CSVエクスポート

1. **かけられたブースト** または **かけたブースト** の「<img src="/user-manual/file-csv-solid.svg" className="inline" height="12" />」を押してください。

   ![Dashboard 004](/user-manual/ja/reference/dashboard-004.png)

2. **かけられたブーストCSVエクスポーター** または **かけたブーストCSVエクスポーター** ダイアログが開きますので、フォームへ次のように入力してください。
   - ①プリセット: エクスポート範囲を現在の日付を元に設定します（7日、2週間、1ヶ月、3ヶ月、6か月、1年）
   - ②開始日付: エクスポート範囲の開始日付（終日が範囲内になります）
   - ③終了日付: エクスポート範囲の終了日付（終日が範囲内になります）
3. 「エクスポート」ボタンを押してください。CSVファイルが **ダウンロード** フォルダに保存されます。

   ![Dashboard 005](/user-manual/ja/reference/dashboard-005.png)

エクスポートされるCSVファイルは以下の様な内容で、1行につきブースト１件が出力されます。

| 列   | タイトル            |  内容                                                                                                                    |
|-----|-----------------|--------------------|-----------------------------------------------------------------------------------------------------------------------|
| A   | 作成日時            |  ブーストが作成された日時                                                                                                          |
| B   | 更新日時            | ブーストが最後に更新された日時                                                                                                       |
| C   | ロック日時           |  ブーストの SecretLock が実行された日時                                                                                             |
| D   | ロック期限           | ブーストの SecretLock の期限（ロック日時 + ブロック数に30をかけた秒数で、実際の時刻はチェーンの状況に依存し、あくまでも目安です）。「返金」の場合は、この時期にアカウントに資産が戻ります。                |
| E   | コンプリート日時        | ブーストを実行者に授与するための SecretProof の実行完了を QUEST が検知した日時。未だ授与されていない場合は空欄。                                                    |
| F   | ID              |  ブースト請求書のID                                                                                                            |
| G   | Txハッシュ（ロック）    | Symbol ブロックチェーンでブーストの SecretLock トランザクションを特定するためのハッシュ値  |
| H   | Txハッシュ（コンプリート） | Symbol ブロックチェーンでブーストを実行者に授与するための SecretProof トランザクションを特定するためのハッシュ値 |
| I   | ステータス           |  ブーストの状態。「支払済」はブースト手数料を支払い SecretLock が実行された状態。「授与」は SecretProof が実行されブーストが相手に着金した状態。「返金」は返金が決定した状態で「ロック期限」時点で資金が戻る状態。 |
| J   | クエスト            |  クエストのタイトル                                                                                                             |
| K   | クエストID          | クエストのID                                                                                                               |
| L   | 実行者             | 実行者の名前                                                                                                                |
| M   | 実行者ID           | 実行者のID                                                                                                                |
| N   | 支払者             | 支払者の名前                                                                                                                |
| O   | 支払者ID           | 支払者のID                                                                                                                |
| P   | ブースト戦略          | ブースト作成時に選択した戦略                                                                                                        |
| Q   | 金額（ロック）         |  SecretLock 実行でロックされた金額(XYM)。ブースト額。                                                                                                |
| R   | 金額（コンプリート）      | SecretProof 実行で相手に着金した金額(XYM)。ブースト額と等しくなる。未だ授与されていない場合は空欄。                                                                             |
| S   | ブースト手数料         | 支払ったブースト手数料(XYM)。尚、SecretLock 実行時に手数料の支払いも行われます。                                                                            |
| T   | トランザクション手数料     | ブースト手数料支払いと SecretLock 実行時に実際支払ったトランザクション手数料合計(XYM)（SecretProof 実行時のトランザクション手数料は QUEST 負担）                                  |
| U   | XYM/JPY ロック時    | ブースト手数料支払い及び SecretLock 実行を QUEST が検知した時点でのレート参考値(JPY)                                                                                     |
| V   | XYM/JPY コンプリート時 | SecretProof 実行を QUEST が検知した時点でのレート参考値(JPY)                                                                                                |

**各種計算式**

- ロックしたXYMの日本円を求めたい場合： `Q * U`
- ブースト手数料の日本円を求めたい場合： `S * U`
- トランザクション手数料の日本円を求めたい場合： `T * U`
- 授与されたXYMの日本円を求めたい場合： `R * V`

## 5. フォロー

**フォロー** には、自分がフォローしているユーザーが一覧されます。

- ①名前でフィルター: 名前でリストをフィルターします（前方一致）。名前でフィルターする場合、ソートは「名前昇順」固定です。
- ②ソート: リストをソートする
  - より新しい: 登録日時の降順でソートします
  - より古い: 登録日時の昇順でソートします
  - より最近クエスト実行: クエスト実行開始日時の降順でソートします。クエストを実行していないとリストに出ません。
  - より最近更新: 更新日時の降順でソートします
  - より以前に更新: 更新日時の昇順でソートします
  - 名前昇順: ユーザー名の昇順でソートします
  - 名前降順: ユーザー名の降順でソートします

![Dashboard 006](/user-manual/ja/reference/dashboard-006.png)

## 6. フォロワー

**フォロワー** には、自分をフォローしているユーザーが一覧されます。

- ①名前でフィルター: 名前でリストをフィルターします（前方一致）。名前でフィルターする場合、ソートは「名前昇順」固定です。
- ②ソート: リストをソートする
  - より新しい: 登録日時の降順でソートします
  - より古い: 登録日時の昇順でソートします
  - より最近クエスト実行: クエスト実行開始日時の降順でソートします。クエストを実行していないとリストに出ません。
  - より最近更新: 更新日時の降順でソートします
  - より以前に更新: 更新日時の昇順でソートします
  - 名前昇順: ユーザー名の昇順でソートします
  - 名前降順: ユーザー名の降順でソートします

リストの行を選択すると、**ユーザーサマリ** が右側に表示されます。
